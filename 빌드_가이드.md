# 빠른 빌드 가이드

## 🚀 가장 빠른 방법: Live Coding (권장)

### 사용 방법:
1. **Unreal Editor를 먼저 실행** (프로젝트 열기)
2. 코드를 수정 (Visual Studio 또는 Cursor에서)
3. **Unreal Editor로 돌아가서 `Ctrl+Alt+F11` 누르기**
4. 에디터를 닫지 않고도 변경사항이 바로 반영됩니다!

### 장점:
- ✅ 에디터를 닫을 필요 없음
- ✅ 가장 빠른 반복 속도
- ✅ 게임 상태 유지 가능

---

## 📝 다른 빌드 방법들

### 방법 1: Visual Studio에서 빌드
1. `WutheringWave.sln` 파일 열기
2. 상단에서 `Development Editor` 선택
3. `빌드` → `솔루션 빌드` (또는 `Ctrl+Shift+B`)
4. 빌드 완료 후 Unreal Editor에서 프로젝트 열기

### 방법 2: 빠른 빌드 스크립트 사용
- **PowerShell**: `.\QuickBuild.ps1` 실행
- **배치 파일**: `QuickBuild.bat` 더블클릭

**주의**: 스크립트 내의 Unreal Engine 경로(`C:\Program Files\Epic Games\UE_5.6`)가 실제 설치 경로와 일치하는지 확인하세요.

### 방법 3: Unreal Editor에서 빌드
1. Unreal Editor에서 프로젝트 열기
2. `Ctrl+Alt+F11` 또는 `파일` → `새 C++ 클래스 컴파일`
3. 에디터가 자동으로 빌드

---

## ⚡ 개발 워크플로우 추천

### 최적의 워크플로우:
```
1. Unreal Editor 실행 (프로젝트 열기)
   ↓
2. Cursor/Visual Studio에서 코드 수정
   ↓
3. Unreal Editor로 전환 (Alt+Tab)
   ↓
4. Ctrl+Alt+F11 (Live Coding)
   ↓
5. 즉시 테스트 가능!
```

### 팁:
- **Live Coding이 작동하지 않을 때**: 에디터를 완전히 닫고 Visual Studio에서 빌드한 후 다시 열기
- **큰 변경사항** (헤더 파일 수정, 클래스 추가 등): 전체 재빌드 필요
- **작은 변경사항** (함수 구현만 수정): Live Coding으로 충분

---

## 🔧 문제 해결

### Live Coding이 안 될 때:
1. 에디터 완전히 종료
2. Visual Studio에서 `Development Editor`로 빌드
3. 에디터 다시 열기

### 빌드 오류가 발생할 때:
1. Visual Studio에서 `빌드` → `솔루션 정리`
2. `빌드` → `솔루션 다시 빌드`
3. 오류 메시지 확인
